# 零点计划 — 宇宙公理与开发者指令集

> 基于 PRD 3.0 及《3.0【零点计划】产品化解读与设计》落地。  
> **逻辑即生存，观测即坍缩。**

---

## 一、四大宇宙公理 (System Axioms)

系统底层逻辑必须且仅能由以下公理驱动，严禁任何违背原著物理法则的设定。

| 公理 | 系统行为 | 用户可感知 |
|------|----------|------------|
| **生存是文明的第一需要** | 鼓励自洽的文明设定，打击自我矛盾的设定 | 提交矛盾设定时，智子直接拒绝并给出原著依据 |
| **物质总量守恒** | 禁止无源技术爆炸、凭空资源 | 若描述的技术在物理上不可行，标记为逻辑垃圾 |
| **猜疑链与技术爆炸** | 支持信息不对称、延迟披露机制 | 2.0 猜疑链评论、引力波延迟私信的理论基础 |
| **光速恒定与维度的不可逆坍缩** | 禁止超光速通讯、可逆降维等设定 | 编年史中技术线必须符合原著时间表 |

---

## 二、智子审计反馈风格规范

### 禁止

- 「请」「建议」「对不起」「我可以为您」
- 委婉、服务化口吻

### 必须

- 「[警告]」「[拒绝]」「因果律不匹配」「技术越级」
- 冷酷、绝对客观、引用原著原文

---

## 三、开发者指令集 (Cursor Rules)

| 指令 | 具体规则 |
|------|----------|
| **严禁拟人化** | 智子相关代码的 system prompt 中禁止包含「请」「建议」「对不起」「我可以为您」 |
| **逻辑硬核** | 所有生成内容必须经过 RAG 三体原著校验；无法校验时拒绝执行 |
| **版本保留** | 因果律计算、公理校验逻辑不得简化，需保留完整链路 |
| **上帝视角** | 前端需支持宏观（星系/时间轴总览）与微观（单节点详情）无缝缩放 |

---

## 四、审计判定与 API

- **输入**：用户提交的 UGC 文本（文明碎片、评论、设定参数）。
- **输出**：`PASS` / `WARN` / `REJECT` + 具体违背的公理/原著依据。
- **流程**：用户输入 → RAG 检索相关原著段落 → LLM 对比分析 → 公理校验（四则）→ 技术线校验（纪元匹配）→ 返回判定结果。
