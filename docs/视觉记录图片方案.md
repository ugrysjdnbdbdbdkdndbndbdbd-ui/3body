# 时间之流「视觉记录」图片方案

时间之流详情页下方的「视觉记录」由编年史事件的 `image_url` 字段驱动，数据来自 `backend/scripts/seed_chronicles.py` 中的 `CHRONICLES`，经 `scripts/seed_db.py` 写入数据库，前端通过 `/api/chronicle/events` 拉取并在 `ChronicleView.vue` 中展示。

## 当前方案：Picsum 稳定图

- **来源**：<https://picsum.photos>
- **格式**：`https://picsum.photos/seed/{seed}/600/400`
- **说明**：同一 `seed` 始终返回同一张图，每条事件使用不同 seed（如 `redcoast1967`、`sophon`、`wallfacer`），保证每条事件有固定且不同的视觉，无需 API key。
- **局限**：图片为随机真实照片，与剧情主题无强关联，仅作占位与氛围填充。

## 替换为自选/自生成图片

若希望视觉记录与事件内容强关联，可按以下步骤操作。

### 1. 准备图片与可访问 URL

- **图床**：将图片上传至项目使用的图床或静态资源服务，得到公网可访问的 URL（如 CDN、对象存储、或 `frontend/public` 下资源对应路径）。
- **推荐尺寸**：约 600×400 或同比例，与当前 Picsum 请求一致，便于详情页展示一致。

### 2. 修改编年史种子数据

在 `backend/scripts/seed_chronicles.py` 的 `CHRONICLES` 中，将对应事件的 `image_url` 改为你的图片 URL，例如：

```python
{
    "era": "危机纪元",
    "year": 1967,
    "title": "红岸·1967",
    # ...
    "image_url": "https://your-cdn.com/redcoast1967.jpg",  # 替换此处
    # ...
}
```

### 3. 重新跑种子（写入数据库）

在项目根目录或 `backend` 下执行：

```bash
python3 scripts/seed_db.py
```

注意：若种子脚本会清空或覆盖编年史表，请确认环境（开发/生产）再执行。

### 4. 前端生效

前端请求 `/api/chronicle/events` 会拿到更新后的 `image_url`，时间之流详情页的「视觉记录」将自动显示新图，无需改前端代码。
