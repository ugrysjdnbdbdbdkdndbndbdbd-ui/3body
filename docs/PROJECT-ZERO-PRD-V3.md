# 📂 【零点计划 v3.0】PRD：破壁行动 (Multimodal & Reality-Sync Expansion)

## 1. 角色与项目全局设定 (Role & Meta Context)
- **AI 角色:** Lead AI Multimodal Architect & Prompt Security Engineer.
- **项目代号:** Project Zero v3.0 (破壁行动).
- **核心目标:** 突破纯文本交互的界限，整合 Vision LLM、TTS (Text-to-Audio)、Real-time API 以及超长上下文 (1M+ Tokens)，将《三体》的绝望感与宏大感渗透进现实世界。
- **架构原则:** 现实即沙盒。要求极高的数据处理效率与极简、冰冷的 UI 表现。

---

## 2. 核心增量模块需求 (Multimodal Modules)

### 模块 A：【视网膜倒计时】 (Retinal Countdown Overlay)
**榨干能力:** 视觉大模型 (Vision LLM) + 图像语义分割
**业务逻辑:**
1. **输入端:** 允许用户上传现实生活的高精度原图（例如由 Sony A7M3 等专业设备拍摄的街景或日常画面）。
2. **AI 解析:** 调用 Vision API，识别画面中的景深、光源与核心锚点。
3. **图像篡改:** 在画面的视觉中心或不经意的角落，无缝渲染一串泛着幽光的“倒计时数字”（如：`1194:12:45:00`）。
4. **智子冷酷侧写:** 返回一段基于画面内容的物理学解构报告。
   - *Prompt 示例:* “分析此图，忽略其现实美学，用智子的视角指出画面中被忽略的微观异常（例如 $0.0003\%$ 的光谱偏移），并宣布倒计时。”

### 模块 B：【三体 V 装具：现实气象同步引擎】 (V-Suit Reality Sync)
**榨干能力:** 外部 API 挂载 (Function Calling)
**业务逻辑:**
1. **位置锚定:** 经用户授权获取其当前物理坐标的实时气象数据（如：菲律宾马尼拉大都会都会区）。
2. **纪元映射:** 编写一个 `Weather_to_Era_Mapper` 算法。
   - 若现实气象 API 返回极端高温、暴雨或强台风（如热带气候典型的极端天气），系统强制将该用户的虚拟文明拖入**【乱纪元】**。
   - 若气候宜人，则进入短暂的**【恒纪元】**。
3. **生存惩罚:** 在乱纪元期间，用户必须在规定时间内上线执行“脱水”指令，否则其文明资产将被系统扣除。这种设计彻底打破了游戏与现实的边界。

### 模块 C：【红岸监听频道】 (Red Coast Radio: 1379)
**榨干能力:** 情感合成 TTS (Text-to-Audio)
**业务逻辑:**
1. **事件触发:** 当后端“多智能体沙盒”中发生光粒打击或文明覆灭事件时，提取其临终日志。
2. **音频渲染:** 调用 TTS 引擎，叠加特定滤镜：
   - 添加深空白噪音、静电干扰声、以及极度冷静或绝望的非自然语调。
3. **前端呈现:** 在 UI 侧边栏常驻一个黑色的“1379 频段”播放器。用户可以戴上耳机，实时监听这些由 AI 生成的、平行宇宙中死去的文明发出的重力波遗言。

### 模块 D：【归零者协议】 (The Returner's Summarization)
**榨干能力:** 百万级超长上下文窗口 (1M+ Token Context)
**业务逻辑:**
1. **触发条件:** 当某条世界线的 UGC 共创文本总量达到算力临界点，触发“二维化”流程。
2. **降维压缩:** 将数十万字的该时间轴所有内容打包，一次性塞入大模型长窗口。
   - *Prompt 示例:* “你现在是归零者。提取这 50 万字文明史诗中最核心的因果链与悲剧，将它们压缩成一首不超过 140 字的古典宇宙墓志铭。不包含任何感情色彩。”
3. **数据焚毁:** 彻底清空数据库中的原本文本，仅在首页的“冥王星墓碑”组件中刻下这 140 个字。

### 模块 E：【思想钢印·越狱攻防】 (Mental Seal Jailbreak)
**榨干能力:** 系统提示词攻防 (Prompt Injection Sandbox)
**业务逻辑:**
1. **目标:** 玩家需要用语言“破壁”一个被设定了绝对信仰（如“逃亡是犯罪”）的 AI 联合国高官。
2. **防护塔:** AI 预设了极强的 System Prompt 锁。
3. **胜利判定:** 若玩家能通过逻辑诡辩、角色扮演等手法，诱导 AI 在回复中输出违背其钢印的言论（如承认“逃亡是唯一的生路”），系统自动判定破壁成功，全服通告。

## 3. 技术实施路径与 API 选型 (Technical Routing)
- **Vision LLM:** DeepSeek-VL 或 GPT-4V 用于处理高分辨率图像上传。
- **Weather API:** OpenWeatherMap API，设置 Webhook 定时轮询活跃用户的现实气象数据。
- **TTS 引擎:** ElevenLabs 或类似具备强情绪控制与音效叠加能力的音频模型。
- **前端架构适配:** 需要在现有的 Next.js 框架中引入 `Web Audio API` 处理实时音频流。
