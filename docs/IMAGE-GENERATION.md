# 三体项目 · Gemini Nano 生图说明

本文档说明使用 **Gemini Nano** 为「人物志」与「编年史」生成图片时的目录结构、生图标准、每条生图 prompt，以及生图后的放置与引用方式。

---

## 1. 图片存放目录

生图完成后请将文件放入以下本地目录（与仓库根目录相对）：

| 用途       | 路径                     | 说明 |
|------------|--------------------------|------|
| 人物志头像 | `assets/figures/`        | 每位人物一张竖版肖像，文件名见下方「人物志 Prompts」 |
| 编年史配图 | `assets/chronicles/`     | 每条编年史一张横版视觉记录图，文件名见「编年史 Prompts」 |

若前端通过静态资源访问，可将上述目录内容复制到 `frontend/public/images/figures` 与 `frontend/public/images/chronicles`，或通过构建脚本复制。

---

## 2. 生图标准

### 2.1 人物志（figures）

| 项目     | 要求 |
|----------|------|
| **尺寸** | 宽 300 px × 高 400 px（3:4 竖版） |
| **比例** | 竖构图，半身或胸像，便于在人物卡片中展示 |
| **格式** | PNG 或 WebP，推荐 WebP（体积更小） |
| **命名** | `{英文名蛇形}.webp`，如 `luo_ji.webp`、`ye_wenjie.webp`。英文名中的空格、括号、连字符转为下划线，全小写。 |

### 2.2 编年史（chronicles）

| 项目     | 要求 |
|----------|------|
| **尺寸** | 宽 600 px × 高 400 px（3:2 横版） |
| **比例** | 横构图，场景/事件视觉记录，适合列表或详情头图 |
| **格式** | PNG 或 WebP，推荐 WebP |
| **命名** | `{事件英文 slug}.webp`，如 `redcoast1967.webp`、`doomsdaybattle.webp`。与下方「编年史 Prompts」表中的「文件名」列一致。 |

### 2.3 风格建议

- **人物志**：写实插画或半写实，三体/近未来科幻感，体现时代（黄金时代/危机纪元等）与身份（面壁者、军人、科学家等），避免过度卡通。
- **编年史**：科幻插画，一张图概括该条事件的关键场景或氛围，色调与情节匹配（压抑、史诗、寂静、绝望等）。

## 5. 使用说明

### 5.1 生图后放置

1. 使用 Gemini Nano 按上表 prompt 生成图片。
2. 将**人物志**图片以「文件名」列中的名称保存到 `assets/figures/`。
3. 将**编年史**图片以「文件名」列中的名称保存到 `assets/chronicles/`。

### 5.2 在前端或后端引用

- **方式 A（静态资源）**：若将 `assets/figures/`、`assets/chronicles/` 复制到 `frontend/public/images/` 下，则前端可引用如 `/images/figures/luo_ji.webp`、`/images/chronicles/redcoast1967.webp`。
- **方式 B（后端 URL）**：若图片由后端或 CDN 提供，在数据库或配置中将人物/编年史的 `image_url` 更新为对应 URL。
- **与现有 seed 的对应**：人物数据见 `backend/scripts/seed_figures.py` 的 `FIGURES`，编年史见 `backend/scripts/seed_chronicles.py` 的 `CHRONICLES`。更新 `image_url` 时，人物可按 `en_name` 对应到上表文件名（如 Luo Ji → `luo_ji.webp`），编年史可按 `title` 对应到上表「事件标题」再取文件名。

### 5.3 重新生图或替换

- 保持**文件名**不变，直接替换同路径下的图片即可，无需改文档或代码。
- 若调整尺寸或格式，请同步更新本文档「生图标准」一节。
