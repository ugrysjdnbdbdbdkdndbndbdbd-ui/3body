# 黑暗森林沙盒 · 商业需求文档 (BRD)

**Dark Forest Sandbox — Business Requirements Document**

> 将「黑暗森林沙盒」升级为 3Body X 特色能力之一，作为后续商业化与产品演进的商业指引。

---

## 1. 文档目的与范围

- **目的**：明确黑暗森林沙盒作为**特色功能**的商业定位、目标用户、价值主张与变现路径，为产品 PRD 与研发优先级提供依据。
- **范围**：商业与市场侧需求；不涉及具体交互与实现细节（见《黑暗森林沙盒-PRD》）。

---

## 2. 市场与机会

### 2.1 品类定位

- **品类**：沉浸式 IP 衍生体验 + 轻量级策略/模拟。
- **差异化**：以《三体》宇宙社会学为规则内核的**可玩化模拟**，而非纯叙事或纯数值游戏。
- **机会点**：
  - 三体 IP 长尾热度与「黑暗森林法则」出圈带来的自然流量。
  - 教育/科普场景：宇宙社会学、博弈论、多智能体系统的直观演示。
  - 硬核科幻与策略玩家的交叉需求：既有世界观深度，又有可操作、可复现的「沙盒」体验。

### 2.2 竞品与差异

| 类型         | 代表                     | 与黑暗森林沙盒的差异 |
|--------------|--------------------------|----------------------|
| 三体衍生游戏 | 官方/授权游戏           | 多为剧情或战斗向；沙盒强调「法则模拟」与可观测、可存档的宇宙演化。 |
| 策略/4X      | 文明、群星等             | 沙盒体量更轻、规则更聚焦「发现—打击—墓志铭」一条链，学习成本低。 |
| 科普/教育    | 博弈论演示、多智能体 demo | 沙盒自带强 IP 叙事与世界观，利于传播与留存。 |

**结论**：将黑暗森林沙盒定位为「三体宇宙社会学第一可玩沙盒」，突出**法则可验证、结果可回溯、世界线可归档**的独特价值。

---

## 3. 目标用户

### 3.1 主用户画像

- **三体原著粉**：希望「亲手验证」黑暗森林法则，而不仅是阅读或观看。
- **策略/模拟爱好者**：喜欢低门槛、高复玩性的小规模多智能体演化。
- **教育/科普场景**：教师、科普 up 主、读书会组织者，用于讲解宇宙社会学或博弈论。

### 3.2 次要用户

- **泛科幻用户**：被「黑暗森林」概念吸引，首次接触 3Body X 的入口。
- **内容创作者**：需要可导出、可复现的「宇宙墓志铭」「世界线对比」作为素材。

### 3.3 用户目标（Jobs to Be Done）

- 观察/操控一个多文明宇宙的演化，理解「技术等级—疑心指数—隐蔽能力」如何导致发现与打击。
- 保存与对比不同世界线（归档/恢复），用于复盘或分享。
- 在有限时间内获得「一局结果」与明确反馈（墓志铭、事件流），满足「一局一故事」的心智。

## 4. 价值主张与特色能力

### 4.1 核心价值主张

- **对用户**：「在几分钟内运行你自己的黑暗森林宇宙，看到谁发现了谁、谁清除了谁，并留下墓志铭。」
- **对产品**：黑暗森林沙盒成为 3Body X 的**标志性可玩模块**，与「智子」「面壁者」「归零者广播」等形成「观看 + 互动 + 模拟」的完整体验链。

### 4.2 特色能力归纳（当前与规划）

- **双端体验**：
  - **Sector E（黑暗森林广场）**：实时 Canvas 沙盒 — 观测者/猎人双模式、光粒打击与探测范围可视化、执剑人威慑小游戏、坐标广播。
  - **Sector Zero v2 · B（黑暗森林沙盒）**：多智能体回合演化、宇宙墓志铭、世界线归档/恢复、事件流导出（JSON/CSV）、与 LLM 网关联动的 `dark-forest.tick`。
- **可沉淀产物**：墓志铭列表、世界线快照、事件流导出 — 支持「复盘、教学、二创」。

上述能力在 PRD 中细化；BRD 仅确立其作为**特色能力**的商业地位。

## 5. 商业化策略

### 5.1 与现有会员体系的关系

- 沿用《COMMERCIAL-PRD》中的身份体系：虫子（免费）、ETO、面壁者、执剑人。
- **免费层（虫子）**：
  - 可玩 Sector E 的宇宙沙盒（观测者模式 + 有限时长或次数）。
  - 可玩 Zero v2 B 的沙盒：默认文明数量与回合间隔，世界线归档条数上限（如 3 条）。
- **付费层（ETO / 面壁者 / 执剑人）**：
  - 猎人模式、更长时长/无限制。
  - 更多世界线槽位、事件流历史条数、导出无水印或带「执剑人」标识。
  - 未来：自定义文明参数、私有剧本、与 A/C/D 模块联动的「跨扇区剧情」（若产品规划）。

### 5.2 变现方式建议

- **主要**：会员订阅（月/年）与执剑人买断中，将「黑暗森林沙盒高阶权益」明确列入权益表。
- **次要**：单次解锁「沙盒扩展包」（如更多文明数量、更长世界线保留期），作为轻量 IAP 或活动奖励。
- **非直接变现**：沙盒作为拉新与留存的钩子 — 首局免费、分享「我的宇宙墓志铭」生成图或链接，带来传播与回流。

### 5.3 定价与包装建议

- 不在本 BRD 中固定价格；建议在 COMMERCIAL-PRD 的「付费解锁功能清单」中新增一行：
  - **Sector E + Zero v2 B（黑暗森林）**：免费版 = 基础沙盒 + 有限归档；付费版 = 猎人模式 + 无限/高额归档 + 导出增强。
- 包装话术突出「运行你的黑暗森林」「书写宇宙墓志铭」「归档与对比世界线」。

## 6. 成功指标与阶段目标

### 6.1 北极星与关键指标

- **北极星**：将黑暗森林沙盒打造成 3Body X 内**使用频次与时长占比前二的互动模块**（与智子/面壁者并列或更高）。
- **关键指标建议**：
  - 沙盒相关页面的 DAU/MAU、人均会话时长、每会话触发「运行/步进」次数。
  - 世界线归档数、事件流导出次数（反映深度使用与二创潜力）。
  - 从沙盒入口进入并完成首单/升级会员的转化率（若已接入支付）。

### 6.2 阶段目标（可随版本迭代调整）

- **Phase 1（当前能力巩固）**：统一 Sector E 与 Zero v2 B 的叙事与入口，在 UI/文档中明确「黑暗森林沙盒」品牌；收集留存与时长数据。
- **Phase 2（能力增强）**：在 PRD 中的「扩展能力」落地后，推出「沙盒扩展包」或将会员权益与新增能力绑定，观测 LTV 与 NPS。
- **Phase 3（生态与传播）**：墓志铭/世界线分享、排行榜或精选「名场面」，形成 UGC 与传播闭环。

---

## 7. 风险与依赖

- **IP 与合规**：确保玩法和文案符合授权与平台规范，避免对「黑暗森林」的曲解或过度暴力描述。
- **技术依赖**：沙盒依赖 LLM 网关时（如 `dark-forest.tick` 服务端逻辑），需考虑降级策略（本地 fallback 已存在）；商业化需保证服务可用性与延迟可接受。
- **资源依赖**：特色功能的长期迭代需要稳定的产品与研发资源分配，建议在路线图中将「黑暗森林沙盒」列为 P0 能力之一。
