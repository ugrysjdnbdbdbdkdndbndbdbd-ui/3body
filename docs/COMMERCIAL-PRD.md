# 3Body X: Commercialization & Identity System PRD
# 三体宇宙模拟器：商业化与身份体系 PRD

> "消灭人类暴政，世界属于三体。" —— 这里的"暴政"指代服务器运维成本。

## 1. 概述 (Overview)

### 1.1 背景与目标
本项目已从单一的视觉展示演化为高负载的物理模拟平台（N-Body 运算、流体动力学、4D 渲染）。为了维持高昂的算力成本并支持后续开发（如 VR/AR 版），需要引入商业化模块。

### 1.2 核心策略：身份即权益
摒弃传统的 "VIP/Pro" 命名，将会员体系深度植入三体世界观。用户付费不仅仅是为了解锁功能，更是为了获得**宇宙社会学的高阶权限**。

## 2. 身份体系 (Identity & Membership)

### 2.1 Level 0: 虫子 (The Bugs)
*   **定位**：免费用户 / 游客。
*   **原著隐喻**：被锁死物理学的人类，只能看到表象。
*   **权益**：
    *   访问所有基础扇区 (Sector A - X)。
    *   基础分辨率渲染 (720p/1080p)。
    *   只读模式：可以观看三体摆、水滴攻击，但无法修改参数。
    *   广播系统：只能接收，不能发送。

### 2.2 Level 1: ETO 成员 (ETO Member)
*   **定位**：月度订阅会员 (￥19.9/月)。
*   **权益**：
    *   **高精模拟**：解锁 Sector T 的 N-Body 混沌运算，支持计算 1000 年后的恒星轨迹。
    *   **4D 视觉**：在 Sector G (四维碎片) 中开启“上帝视角”透视模式。
    *   **专属徽章**：用户名旁显示 ETO 黄金徽章。
    *   **无广告**：移除智子视网膜干扰（模拟广告位）。

### 2.3 Level 2: 面壁者 (The Wallfacer)
*   **定位**：年度订阅会员 (￥199/年)。
*   **权益**：
    *   **包含 Level 1 所有权益**。
    *   **参数修改权 (God Mode)**：
        *   Sector L: 修改光速常量 (c)。
        *   Sector W: 自定义水滴攻击路径与速度。
    *   **咒语实验**：拥有向宇宙广播恒星坐标的权限（每月 1 次）。
    *   **专属皮肤**：解锁“二向箔金”与“水滴银”UI 主题。

### 2.4 Level 3: 执剑人 (The Swordholder)
*   **定位**：终身买断 / 赞助者 (￥648)。
*   **权益**：
    *   **全站解锁**。
    *   **归零者权限**：在 Sector X 中拥有自定义广播内容的权利（向全服用户推送消息）。
    *   **源码访问**：获取部分核心算法（如 4D 投影、三体运动引擎）的源码阅读权限。
    *   **实体周边**：赠送实体“黑暗森林法则”金属铭牌。

## 3. 账户注册与登录 (Account System)

### 3.1 登录流程 (The Login)
*   **支持方式**：
    *   **物理密钥 (Email/Phone)**：常规验证码登录。
    *   **视网膜验证 (FaceID/TouchID)**：移动端生物识别登录（文案包装）。
    *   **三体游戏入口 (V-Suit)**：通过 Web3 钱包 (Metamask) 登录（保留接口）。

### 3.2 注册引导 (Onboarding)
*   用户首次进入，需通过“三体游戏第一关”测试（简单的物理学问答），方可注册 ETO 账号。
*   答错者被判定为“无科学素养”，仅可作为“虫子”浏览。

## 5. 支付链路 (Payment Flow)

### 5.1 支付网关 (Gateway)
*   **国内**：支付宝 (Alipay)、微信支付 (WeChat Pay)。
*   **国际**：Stripe (Credit Card), PayPal。
*   **加密货币**：接受 BTC/ETH 捐赠（作为“逃亡基金”）。

### 5.2 交互流程
1.  **触发**：用户点击“修改光速”或“高清导出”。
2.  **拦截**：智子弹出提示框 —— *"主不在乎，但服务器在乎。我们需要资源。" / "The Lord does not care, but the server does."*
3.  **展示**：弹出全息风格的会员权益对比表（ETO 招募简章）。
4.  **支付**：调用原生支付组件。
5.  **回调**：支付成功，播放“统帅确认”音效，界面解锁高阶功能，UI 变为金色/银色。

## 6. 技术架构 (Technical Architecture)

### 6.1 用户系统 (Auth)
*   使用 **Supabase Auth** 或 **Clerk** 托管用户身份。
*   用户数据结构：
    ```typescript
    interface UserProfile {
      id: string;
      role: 'bug' | 'eto' | 'wallfacer' | 'swordholder';
      exp: number; // 贡献值
      unlocked_sectors: string[];
      inventory: string[]; // 收藏的二向箔画作
    }
    ```

### 6.2 订单系统 (Order)
*   后端服务（Node.js/Go）处理订单创建与 Webhook 回调。
*   数据库记录：`orders` (订单流水), `subscriptions` (订阅状态)。

### 6.3 权益控制 (Permission)
*   前端：`usePermission()` Hook，控制 UI 显隐。
*   后端：API 鉴权，防止通过抓包调用高阶计算接口。

## 7. 风险与合规 (Risk & Compliance)
*   **文案合规**：避免使用过于激进的政治隐喻，确保“反人类暴政”等原著台词在娱乐语境下使用。
*   **退款政策**：提供 7 天内无理由退款（称为“冬眠唤醒保障”）。
*   **隐私保护**：严格遵守 GDPR，不收集用户生物信息（仅本地调用 FaceID）。
