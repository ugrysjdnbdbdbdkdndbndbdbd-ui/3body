# 📂 产品需求文档 (PRD)：【零点计划】—— 智子终端与因果律模拟器

**项目代号：** Project Zero (零点计划)

**目标：** 构建一个严格遵守《三体》宇宙公理的、由用户共创驱动的实时演化宇宙实验室。

**核心哲学：** 逻辑即生存，观测即坍缩。

---

## 1. 宇宙公理 (System Axioms)

整个系统的底层逻辑必须且仅能由以下公理驱动，严禁任何违背原著物理法则的设定：

1. **生存是文明的第一需要。**
2. **文明不断增长和扩张，但宇宙中的物质总量保持不变。**
3. **猜疑链与技术爆炸。**
4. **光速恒定与维度的不可逆坍缩。**

---

## 2. 核心功能模块 (Core Modules)

### 2.1 智子审计核心 (Sophon Logic Engine)

这是系统的“大脑”，负责维护宇宙的逻辑一致性。

- **因果律扫描：** 每一条用户提交的“文明碎片”（UGC 内容）必须经过智子的实时审计。
- **判定机制：** 利用 RAG 检索原著全集，分析内容是否违背物理常数或技术线。
- **反馈风格：** 智子的反馈应是冷酷且绝对客观的。例如：“[警告]：在当前纪元描述曲率驱动属于技术越级，因果律不匹配，该碎片已标记为‘逻辑垃圾’。”

### 2.2 动态因果律时间轴 (Causality Timeline)

- **概率云状态：** 未被大规模观测（互动）的时间轴分支处于叠加态。
- **观测者效应：** 当某一文明碎片的互动热度超过阈值，该路径发生“坍缩”，正式合并入全服主时间轴。
- **因果回溯：** 用户可以沿时间轴向上滑动，查看每一个关键历史节点的变数来源。

### 2.3 文明碎片共创 (Civilization Fragments)

- **外传补完：** 鼓励用户填充原著中的“留白”纪元（如大低谷时期的地下城市生活、掩体纪元的太空社会学）。
- **组件化创作：** 用户提交的不仅是文字，还有技术参数、社会公理、人物行为包。这些组件会像齿轮一样在模拟器中相互咬合。

---

## 3. 全局动态交互 (Global Interaction)

### 3.1 黑暗森林过滤 (The Dark Forest Filter)

- **清理机制：** 逻辑漏洞严重或违背公理的内容会被赋予“暴露坐标”。
- **光粒/二向箔打击：** 系统周期性自动执行“清理”。被打击的内容会在 UI 上呈现视觉破碎或降维效果，并最终归入“重力波档案馆”。

### 3.2 全服共振事件 (Global Resonance)

- **宏观触发：** 当全服用户的集体行为触发特定条件（如：对三体文明的敌意值达到临界，或引力波广播被激活），系统进入“共振状态”。
- **视觉坍缩：** 全站 UI 实时降维。
    - *3D 模式（正常检索） -> 2D 模式（纯文本/平面化） -> 1D 模式（代码流）。*

---

## 4. UI/UX 视觉规范 (Visual Specification)

- **界面风格：** ETO 远程终端 / 掩体纪元操控台。
- **主色调：** * `#001A00` (深空黑)
    - `#00FF41` (智子屏显绿)
    - `#FF0000` (逻辑崩溃预警红)
- **交互动效：** 所有加载动作均模拟“智子从十一维展开”或“维度收缩”。

---

## 5. 开发者指令集 (Cursor/Developer Rules)

在进行后续开发时，Cursor 必须严格遵守以下指令：

> **严禁拟人化：** 智子接口不得使用任何带有情绪或服务性质的词汇（如“请”、“建议”、“对不起”）。**逻辑硬核：** 所有的生成内容必须经过 RAG 的三体原著校验，如果用户输入不合逻辑，直接拒绝执行。**版本保留：** 所有的迭代更新必须保留历史逻辑链路，不得简化核心公理的计算过程。**上帝视角：** 前端展示必须提供宏观（星系级）与微观（个体碎片级）的无缝缩放体验。
>

## 6. 未来展望 (Transcendence)

- **宇宙归零：** 当全服逻辑累积到一定程度，系统将开启“归零任务”，试图让宇宙重回十维。
- **重力波遗言：** 允许用户在文明覆灭前留下 140 字的重力波广播，作为全服永恒保留的“墓碑数据”。
