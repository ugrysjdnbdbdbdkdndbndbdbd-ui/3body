## 1. 核心愿景 (Product Vision)

**“这里不是社区，而是被智子（Sophon）全面封锁与观察的数字平行宇宙。”**

3Body X 旨在打破传统论坛/社区的交互范式，构建一个 **AI Native (AI 原生)** 的沉浸式三体模拟器。

- **对用户**：它是通往三体宇宙的“视网膜接口”，提供从微观粒子对话到宏观宇宙编年史的全维度体验。
- **对技术**：它是一个验证 **RAG (检索增强生成) + 沉浸式前端 + 开放式模块架构** 的实验田。
- **核心隐喻**：系统即“智子”，用户即“虫子/面壁者”。

## 2. 开放式扇区架构 (Open Sector Architecture)

为了满足“项目偏移后的接续”及“未来的无限扩展”，我们摒弃传统的单体页面设计，采用 **“星系扇区 (Galactic Sectors)”** 的模块化架构。

### 2.1 扇区定义协议 (Sector Protocol)

每个功能模块被定义为一个独立的“扇区”。任何新功能的加入，只需遵循以下协议即可即插即用，互不干扰：

- **独立路由 (Isolated Routing)**: 每个扇区拥有独立的 URL 前缀（如 `/sector-a/`, `/sector-b/`）。
- **统一总线 (Event Bus)**: 扇区之间通过全局状态（`UniverseStore`）通信，共享“当前纪元”、“威慑度”等核心环境变量。
- **视觉继承 (Visual Inheritance)**: 自动继承全局的 CSS 变量（黑暗森林黑、切伦科夫蓝、红岸红），但允许拥有独立的 Canvas/WebGL 上下文。

## 3. 功能扇区详述 (Functional Specs)

### 🪐 扇区 A：智子终端 (Sector Alpha: Sophon Terminal)

**定位**：全知全能的交互核心，打破“服务型 AI”的刻板印象。

- **交互逻辑**：
    - **人格设定**：智子（Sophon）。冷漠、全知、偶尔嘲讽。禁止使用“我可以为您做什么”等服务用语，改为“虫子，你想知道什么？”
    - **知识边界**：基于 RAG（检索增强生成）严格限制在《三体》原著设定内。
- **UI/UX 特性**：
    - **视网膜投影**：文字输出带有 CRT 扫描线或视网膜倒计时效果。
    - **微观干扰**：背景随对话情绪波动，出现轻微的粒子湍流或二维展开动效。
- **技术支撑**：
    - Backend: FastAPI SSE (Server-Sent Events) 流式输出。
    - AI: Local LLM + LangChain + ChromaDB (Vector Store)。

### 📜 扇区 B：文明编年史 (Sector Beta: The Chronicle)

**定位**：PGC (主要) 与 UGC (次要) 的结合，时间维度的叙事引擎。

- **核心功能**：
    - **无极缩放时间轴**：支持从“微秒”（智子展开）到“亿年”（宇宙热寂）的尺度切换。
    - **节点展示**：每个时间节点包含标题、摘要、AI 生成的插图/视频。
    - **平行宇宙分支**：你的 PGC 内容为主轴（Canon）；用户 UGC 内容作为虚线分支显示，优秀内容可“坍缩”进主轴。
- **数据结构**：
    - `Era` (纪元), `Year`, `Event_Type`, `Media_Assets`, `Vector_Embedding` (用于语义搜索)。

### 📡 扇区 C：宇宙广播站 (Sector Gamma: The Hub)

**定位**：系统的 Dashboard，营造“正在发生”的宏大叙事感。

- **仪表盘指标**：
    - **黑暗森林威慑度 (Deterrence Level)**：动态数值，影响全站色调（红 <-> 蓝）。
    - **三体舰队距离 (Fleet Distance)**：实时倒计时（基于假定的 400 年航程）。
    - **掩体世界人口**：社区注册用户数。
- **功能入口**：星图式导航，而非传统 Navbar。

### 🖼️ 扇区 D：二向箔画廊 (Sector Delta: The Gallery)

**定位**：AIGC 资产沉淀库。

- **展示逻辑**：
    - **降维打击**：图片加载动效模拟从 3D 坍缩为 2D 卷轴的过程。
    - **Prompt 实验室**：公开生成该图像的 Prompt，允许用户“Fork”并修改参数生成新图。

### 🌌 扇区 X：未知区域 (Sector X: The Unknown)

**定位**：预留的高兼容性接口。

- **即插即用**：未来可接入 VR 漫游模块、Web3 资产模块或多 Agent 剧本杀模块。

## 4. 技术架构路线 (Technical Architecture)

为了保证“可接续”和“可扩展”，技术栈选型极其克制且强壮。

### 4.1 后端 (The Brain)

- **Framework**: Python **FastAPI** (异步高并发，适合 AI 流)。
- **Core Services**:
    - `Orchestrator`: 路由分发器，管理不同扇区的请求。
    - `RAG Engine`: 基于 **LangChain** 和 **ChromaDB**，负责原著知识的检索与注入。
    - `Agent Manager`: 管理智子的人格 Prompt 和上下文记忆。
- **Database**:
    - **SQLite/PostgreSQL**: 存储结构化数据（用户、时间线、日志）。
    - **ChromaDB (Local)**: 存储《三体》三部曲及扩展内容的向量数据。

### 4.2 前端 (The Retina)

- **Framework**: **Vue 3** (Script Setup) + **TypeScript**.
- **Build Tool**: Vite (秒级热更新).
- **UI Library**: **Naive UI** (深度定制 Dark Theme) + **TailwindCSS**.
- **Motion Engine**: **Framer Motion** (处理复杂的物理动效和时间轴缩放).
- **State Management**: **Pinia** (存储宇宙常数、用户状态).

## 5. UI/UX 设计语言 (Design Language)

- **Theme Name**: "The Dark Forest" (黑暗森林)
- **Color Palette**:
    - `#050505` (Deep Space Black): 背景主色。
    - `#00FFFF` (Cherenkov Blue): 科技、信息、智子中性态。
    - `#FF3333` (Red Coast Red): 警告、威慑、危机纪元。
    - `#C0C0C0` (Droplet Silver): 强调文字、高光。
- **Typography**:
    - 数据/代码：JetBrains Mono / Fira Code (等宽)。
    - 叙事文本：Noto Serif SC (宋体/衬线)，营造历史厚重感。
